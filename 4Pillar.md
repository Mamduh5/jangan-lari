The 4 pillars, rewritten into an implementable plan
Pillar 1 — Earlier excitement and stronger pacing

This is the first pillar even if it was not the flashiest one before.

Objective

Make the first 2–4 minutes reliably interesting.

Why first

Your analysis already showed the better content is backloaded and weaker runs may die before the game becomes fun. So before adding more content, the current content must arrive earlier and with better rhythm.

What to change
shorten the “plain early game” period
bring the first meaningful elite/encounter reward earlier
increase odds that a run gets at least one real build-defining moment early
reduce stretches where the player is only orbiting and vacuuming XP
Concrete design targets
first “special moment” should happen much earlier than now
first 3 minutes should include:
at least one pressure spike
at least one reward spike
at least one build-direction choice
stage progression should feel like it is escalating, not idling
Low-risk implementation ideas
earlier elite timing
earlier special wave timing
tighter early enemy pool ramp
slightly earlier weapon/signature access
clearer reward banners and payoff text in HUD
Success condition

A normal run should feel like it “started” quickly, not after several minutes.

Pillar 2 — Stronger build identity

This should be the second pillar.

Objective

Each run becomes a recognizable build archetype, not generic stat inflation.

Why second

This is the biggest answer to “balanced but boring.”
But it should be built on top of improved pacing, otherwise the player still may not live long enough to enjoy it.

Design rule

Prefer upgrades that change:

behavior
targeting pattern
damage delivery style
positioning incentives
survival strategy

Over upgrades that only increase numbers.

Target build families

Do not try to build 20 archetypes. Start with 4–6 strong ones.

Good candidates from your inspiration:

burn / detonation
chain / lightning
orbit / zone control
crit / burst
summon / companion
close-range aggression / risk-reward melee zone
Implementation direction

Use the systems you already have:

weapon signatures
weapon stat patches
breakthrough logic
weapon unlock progression

Expand from “generic power pool” toward:

more weapon-linked upgrades
more synergy-linked upgrades
some conditional upgrades that only appear when a build direction exists
Important constraint

Do not try to solve this with giant new subsystems first.
You can get a lot of value by:

increasing signature coverage
adding 1–2 meaningful branch upgrades per weapon/family
reducing dominance of generic always-good picks
Success condition

After 5 runs, the player can describe them as:

“that fire explosion run”
“that disc/orbit run”
“that chain lightning run”

Instead of:

“same run but better stats.”
Pillar 3 — Mid-run event beats

This is the third pillar and where the game starts feeling more alive.

Objective

Give runs memorable interruptions and decisions besides level-up and boss timing.

Why third

This adds life and surprise, but if pacing and build identity are weak first, events become decoration instead of real excitement.

Design rule

Events should create:

movement decisions
risk/reward tension
build-shaping moments
tempo change
Best first event types

Start with only 2–3 event systems, not many.

Best candidates:

Cursed altar
strong upside
real downside
build-defining choice
Challenge wave
short timed pressure spike
reward on clear/survive
breaks normal flow
Treasure target / roaming reward enemy
forces pursuit
introduces route pressure
makes map space matter temporarily

These are strong because they add action without requiring a massive map/system rewrite.

Avoid for now
full shop systems
elaborate objective chains
big map-interaction framework
many overlapping event types
Success condition

A run includes at least 1–2 moments where the player thinks:
“Do I commit to this risk right now?”

Pillar 4 — Better enemy role contrast and encounter composition

This is important, but it should come after the game already has better pacing and build identity.

Objective

Make combat texture come from enemy combinations, not just enemy quantity.

Why fourth

The analysis was right: you already have decent enemy role potential. The issue is not only missing enemies; it is that encounter composition is not authored enough yet.

Design rule

Think in battlefield problems, not spawn bags.

Examples:

chasers + ranged punishers
tank blockers + flankers
summoner protected by pressure units
hazard creator plus pursuit pack
fast fragile rush mixed with slow area denial
What to add first

Not a huge new roster. First improve composition with existing or lightly expanded roles.

Best additions if needed:

summoner/supporter
blocker/tank
hazard layer/on-death denial
pressure flanker
Implementation direction

Push SpawnDirector toward:

themed wave groups
clearer role combos
short encounter scripts
less purely random mixed spam
Success condition

Players start noticing “what kind of wave this is,” not just “more enemies came.”
